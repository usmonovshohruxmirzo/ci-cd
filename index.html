‚öôÔ∏è Core Concepts Before Starting

  • on: ‚Üí Defines the event that triggers the workflow. Examples:

    • on: push ‚Üí run when you push code.
    • on: pull_request ‚Üí run when PR is opened/updated.
    • on: workflow_dispatch ‚Üí manual trigger.

  • jobs: ‚Üí A workflow is made of one or more jobs.

    • Each job runs on a runner (like ubuntu-latest).
    • Jobs can run in parallel or depend on each other (needs:).

  • steps: ‚Üí Each job has steps (a sequence of commands/actions).

  • uses: ‚Üí Run a prebuilt GitHub Action (from Marketplace). Example: actions/checkout@v3.

  • run: ‚Üí Run custom shell commands (bash on Linux runners).

  • needs: ‚Üí Makes jobs depend on other jobs (controls order).

  • with: ‚Üí Pass input values to actions.

üîπ Stage 2: Run Jobs (Node.js or Python)

üëâ Goal: Run scripts/tests.

üîπ Stage 3: Matrix Builds

üëâ Goal: Test across multiple versions.

üìå .github/workflows/03-matrix.yml

name: Matrix Build

on: [push]

jobs:
  build:
    runs-on: ubuntu-latest
    strategy:              # Define test matrix
      matrix:
        node: [16, 18, 20] # Node.js versions
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: ${{ matrix.node }}
      - run: npm install
      - run: npm test

‚úÖ Runs tests on Node.js 16, 18, 20 simultaneously.

üîπ Stage 4: Lint + Test + Build (Pipeline Stages)

üëâ Goal: Create real CI pipeline.

üìå .github/workflows/04-pipeline.yml

name: CI Pipeline

on: [push]

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - run: npm install
      - run: npm run lint

  test:
    runs-on: ubuntu-latest
    needs: lint     # Run only after lint succeeds
    steps:
      - uses: actions/checkout@v3
      - run: npm install
      - run: npm test

  build:
    runs-on: ubuntu-latest
    needs: test     # Run only after test succeeds
    steps:
      - uses: actions/checkout@v3
      - run: npm install
      - run: npm run build

‚úÖ Enforces Lint ‚Üí Test ‚Üí Build order.

üîπ Stage 5: Cache Dependencies

üëâ Goal: Optimize workflow speed.

üìå .github/workflows/05-cache.yml

name: CI with Cache

on: [push]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Cache node_modules
        uses: actions/cache@v3
        with:
          path: node_modules
          key: ${{ runner.os }}-node-${{ hashFiles('package-lock.json') }}
          restore-keys: |
            ${{ runner.os }}-node-

      - run: npm install
      - run: npm test

‚úÖ Dependencies won‚Äôt reinstall every time.

üîπ Stage 6: Deploy (GitHub Pages Example)

üëâ Goal: Continuous Deployment.

üìå .github/workflows/06-deploy.yml

name: Deploy to GitHub Pages

on:
  push:
    branches: [ "main" ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: 18
      - run: npm install
      - run: npm run build
      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist

‚úÖ Every push ‚Üí deploys site to GitHub Pages.
